# Contributor Stakeholders

## Purpose

Defines contributor personas, needs, barriers, and engagement strategy for the open-source community.

## Contributor Types

### 1. Code Contributors
- Backend (Python/FastAPI)
- Frontend (TypeScript/React/Electron)
- Motivated by: Resume building, learning, solving own needs

### 2. Documentation Contributors
- Technical writers
- Improve onboarding and clarity

### 3. QA/Testing Contributors
- Manual testing and test case writing

### 4. Translators (Future)
- UI/content localization

## Contributor Needs

1. **Clear Guidelines**: See [CONTRIBUTING.md](../../../CONTRIBUTING.md), [CLAUDE.md](../../../CLAUDE.md)
2. **Responsive Maintainers**: Target <48hr response time
3. **Recognition**: Changelog mentions, credits
4. **AI-Assisted Coding**: Explicitly encouraged
